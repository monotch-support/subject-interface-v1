    Architecture Layers

    The Subject Interface uses a dual-layer architecture that separates management operations from data streaming.

    Layer Interaction

    The two layers work together to provide a C-ITS communication solution:

    JSON-REST API Layer

    The foundational layer provides lifecycle management and administrative control through web service interfaces.

    Core Characteristics

    Characteristic Description
    Protocol JSON-REST principles
    Access Control Role-based permissions
    Management Resource lifecycle operations
    Multi-tenancy Cross-domain and jurisdiction support

    Primary Functions

    Function Purpose Capabilities
    Session Lifecycle Manage streaming sessions Session creation with protocol selection, real-time monitoring, dynamic updates, graceful termination
    Authorization & Tokens Control access and credentials Role-based authorization, token generation/rotation, permission scopes, multi-token support
    Subject Registration Register and configure entities Identifier assignment, type specification, domain association, metadata
    Administrative Monitoring Access logs and status Historical session data, connection logs, system health status

    TCP Streaming Protocol Layer

    The data exchange layer optimized for ultra-low latency, high-volume communication.

    Core Characteristics

    Characteristic Description
    Implementation Native TCP streaming
    Data Exchange Continuous asynchronous bidirectional
    Payload Design Message-agnostic, supports any type
    Monitoring Built-in time synchronization and health tracking

    Performance Features

    Feature Benefits
    Ultra-Low Latency Real-time communication, minimal protocol overhead, direct TCP sockets, efficient frame transmission
    High Throughput High-frequency messages, concurrent streams, efficient buffers, backpressure handling
    Minimal Overhead Compact datagram headers, binary protocol, no unnecessary layers, C-ITS optimized
    Scalable Architecture Single/multiple participants, horizontal scaling, load distribution, efficient connections

    Protocol Features

    Feature Implementation
    Version Management Protocol negotiation at connect, future compatibility, graceful mismatch handling, version-specific features
    Frame-Based Communication Structured datagrams with integrity checking, fixed headers, variable payloads, boundary detection
    Time Synchronization Clock difference detection, regular timestamp exchanges, drift termination, payload age tracking
    Security Integration Optional TLS v1.2, per-session configuration, token-based auth
