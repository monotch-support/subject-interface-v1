    Session Logging

    Session logging endpoints provide access to historical session information and operational data for monitoring and analysis purposes. These endpoints support troubleshooting, operational analysis, and compliance reporting.

    Overview

    Session logging allows you to:

    • Retrieve historical session logs within specified time ranges
    • Get detailed information about specific session instances
    • Analyze session performance and operational metrics
    • Monitor session patterns and usage trends

    All log data is automatically filtered based on your API key's authorization context.

    Available Operations

    List Session Logs

    GET /v1/sessionlogs

    Retrieve session logs within a specified time range with automatic filtering based on your API key's authorization context.

    Required Roles: TLC Admin/Analyst, Broker Admin/Analyst, Monitor Admin/System

    Required Query Parameters:

    • from - Start time in ISO8601 format (e.g., "2024-01-15T00:00:00Z")
    • until - End time in ISO8601 format (e.g., "2024-01-15T23:59:59Z")

    Response: Array of session log objects with complete session history and metrics.

    Example Request: 

    GET /v1/sessionlogs?from=2024-01-15T00:00:00Z&until=2024-01-15T23:59:59Z

    Retrieve Specific Session Log

    GET /v1/sessionlogs/{token}

    Get detailed information about a specific session's complete lifecycle and performance data.

    Required Roles: TLC Admin/Analyst, Broker Admin/Analyst, Monitor Admin/System

    Response: Single session log object with details including:

    • Session configuration and parameters
    • Complete timeline of session events
    • Performance metrics and statistics
    • TLC scope history and changes
    • Connection and termination details

    Session Log Data Structure

    Session logs contain information about each session's lifecycle:

    Basic Session Information

    • Token: Unique session identifier
    • Domain: Session domain context
    • Account: Associated account identifier
    • Type: Session type (TLC, Broker, Monitor)
    • Protocol: Communication protocol used
    • Details: Session-specific configuration (stored as JSON string)

    Temporal Data

    • Created: Session creation timestamp (Unix epoch milliseconds)
    • Connected: First successful connection timestamp (Unix epoch milliseconds)
    • Ended: Session termination timestamp (Unix epoch milliseconds)
    • End Reason: Reason for session termination

    Network Information

    • Remote Address: Client IP address that connected to the session

    TLC Scope History

    For sessions with dynamic TLC scope, logs include:

    • Timestamp: When scope change occurred (Unix epoch milliseconds)
    • Scope: The scope level at that time
    • TLC Identifier: Specific TLC identifier affected

    Time Range Queries

    Query Parameters

    Both from and until parameters are required and must be in ISO8601 format:

    GET /v1/sessionlogs?from=2024-01-15T08:00:00Z&until=2024-01-15T18:00:00Z

    Time Range Guidelines

    • Maximum Range: Query time ranges should be reasonable to avoid performance issues
    • Time Zone: Always use UTC (Z suffix) for consistent results
    • Precision: Timestamps support millisecond precision
    • Inclusivity: Time ranges are inclusive of both start and end times

    Example Time Range Patterns

    Single Day: 

    from=2024-01-15T00:00:00Z&until=2024-01-15T23:59:59Z

    Business Hours: 

    from=2024-01-15T08:00:00Z&until=2024-01-15T18:00:00Z

    Last Hour: 

    from=2024-01-15T14:00:00Z&until=2024-01-15T15:00:00Z

    Use Cases

    Troubleshooting

    Session logs help diagnose issues:

    • Connection Problems: Review connection and termination patterns
    • Performance Issues: Analyze session duration and end reasons
    • Configuration Issues: Examine session details and TLC scope changes
    • Network Issues: Investigate remote address patterns and connection timing

    Analytics and Reporting

    Generate insights from historical data:

    • Usage Patterns: Identify peak usage times and session frequency
    • Performance Trends: Track session success rates and duration patterns
    • Resource Utilization: Monitor TLC usage across different sessions
    • Compliance Reporting: Generate audit trails for regulatory requirements

    Operational Monitoring

    Monitor system health and performance:

    • Session Success Rates: Track ratio of successful to failed sessions
    • Duration Analysis: Identify unusually long or short sessions
    • Error Pattern Detection: Spot recurring termination reasons
    • Capacity Planning: Understand session load patterns

    Data Retention

    Session log data retention policies:

    • Availability: Logs are available for a defined retention period
    • Coverage: All session lifecycle events are captured
    • Performance: Recent logs have faster query performance
    • Archival: Older logs may be archived with longer retrieval times

    Error Responses

    Common error scenarios:

    • 400 Bad Request - Invalid time range format or missing required parameters
    • 401 Unauthorized - Invalid or missing authorization token
    • 403 Forbidden - Insufficient permissions for log access
    • 404 Not Found - Session token not found or not accessible within your context

    Best Practices

    Query Optimization

    • Reasonable Time Ranges: Avoid excessively large time ranges that may impact performance
    • Specific Filtering: Use the most specific time ranges possible for your use case
    • Pagination Awareness: Consider implementing pagination for large result sets
    • Error Handling: Implement proper retry logic for network or service issues

    Data Analysis

    • Trend Analysis: Look for patterns over time rather than single events
    • Correlation Analysis: Compare logs with external events for root cause analysis
    • Baseline Establishment: Understand normal patterns to identify anomalies
    • Regular Review: Establish regular log review processes for proactive monitoring

    Security and Privacy

    • Access Control: Ensure log access is limited to authorized personnel
    • Data Sensitivity: Be aware that logs may contain sensitive operational information
    • Audit Trail: Maintain records of who accessed logs and when
    • Data Export: Follow organizational policies for log data export and storage

    For complete API specifications and interactive testing, see the API Reference.
