    TLC Registration

    TLC registration endpoints manage the registration and configuration of subjects within your domain. These endpoints handle the lifecycle of TLC entities that participate in streaming sessions.

    Note: In this API, "TLC" refers to any data-producing entity in the C-ITS ecosystem. This includes traffic controllers, sensors, detection systems, and other connected devices. The term "Subject" is used interchangeably with "TLC" to reflect the broader scope of entities that can participate in data streaming sessions.

    Overview

    TLC registration allows you to:

    • Register new TLCs (subjects) within your domain
    • List and retrieve TLCs you have access to
    • Remove TLCs from the system

    All operations are automatically scoped to your domain and account based on your API key's authorization context.

    Available Operations

    List TLCs

    GET /v1/tlcs

    Retrieve TLCs with automatic filtering based on your API key's authorization context.

    Required Roles: TLC Admin/Analyst, Broker Admin/System/Analyst, Monitor Admin/System

    Response: Array of TLC objects with UUID, identifier, domain, account, and type.

    Create TLC

    POST /v1/tlcs

    Register a new TLC (subject) within your domain and account context.

    Required Roles: TLC Admin

    Request Body: 

    {
  "identifier": "tlc12345"
}

    Key Points:

    • Domain, account, and type are automatically set from your API key's context
    • Identifier must be exactly 8 characters (lowercase alphanumeric)
    • Identifier must be unique within your domain
    • Type defaults to "TCPStreaming" (the only supported type)

    Retrieve Specific TLC

    GET /v1/tlcs/{uuid}

    Get detailed information about a specific TLC.

    Required Roles: TLC Admin/Analyst, Broker Admin/System/Analyst, Monitor Admin/System

    Response: Single TLC object with complete details.

    Delete TLC

    DELETE /v1/tlcs/{uuid}

    Remove a TLC from the system. Any active sessions using this TLC will be terminated.

    Required Roles: TLC Admin

    Response: 204 No Content on successful deletion.

    Important: TLC deletion is irreversible and immediately terminates related sessions.

    TLC Identifiers

    Format Requirements

    TLC identifiers must follow these rules:

    • Exactly 8 characters in length
    • Lowercase alphanumeric characters only (a-z, 0-9)
    • Unique within your domain
    • Pattern: ^[a-z0-9]{8}$

    Examples of Valid Identifiers

    • tlc00001 - Simple numeric sequence
    • main0001 - Descriptive with numbers
    • ctrl5678 - Abbreviated controller reference
    • zone12ab - Zone-based identifier

    Examples of Invalid Identifiers

    • TLC00001 - Contains uppercase letters
    • tlc_001 - Contains underscore
    • tlc001 - Too short (only 6 characters)
    • verylongid - Too long (10 characters)

    TLC Types

    Currently, only one TLC type is supported:

    • TCPStreaming - Standard TCP streaming protocol implementation

    The type field is automatically set and cannot be modified, as it represents the only supported communication protocol.

    Integration with Sessions

    TLCs are referenced in session creation:

    Singleplex Sessions

    Reference a single TLC: 

    {
  "domain": "your-domain",
  "type": "TLC", 
  "protocol": "TCPStreaming_Singleplex",
  "details": {
    "securityMode": "NONE",
    "tlcIdentifier": "tlc12345"
  }
}

    Multiplex Sessions

    Reference multiple TLCs: 

    {
  "domain": "your-domain",
  "type": "TLC",
  "protocol": "TCPStreaming_Multiplex", 
  "details": {
    "securityMode": "NONE",
    "tlcIdentifiers": ["tlc12345", "tlc67890"]
  }
}

    Best Practices

    Naming Conventions

    • Use consistent prefixes for related TLCs
    • Include location or function indicators where helpful
    • Avoid generic names that don't provide context
    • Document your naming scheme for team consistency

    Lifecycle Management

    • Register TLCs before creating sessions that reference them
    • Clean up unused TLC registrations periodically
    • Monitor TLC usage through session logs

    Security Considerations

    • Use TLC-scoped authorizations when users only need specific TLC access
    • Regularly audit TLC access patterns
    • Remove TLCs that are no longer operational
    • Monitor for unauthorized TLC registration attempts

    Error Responses

    Common error scenarios:

    • 400 Bad Request - Invalid identifier format or duplicate identifier
    • 401 Unauthorized - Invalid or missing authorization token
    • 403 Forbidden - Insufficient permissions for the operation
    • 404 Not Found - TLC UUID not found or not accessible

    Operational Patterns

    Batch Registration

    For multiple TLC registrations, create them individually but consider:

    • Consistent naming schemes
    • Grouping related TLCs logically
    • Documentation of TLC purposes
    • Verification of successful registration

    Migration Scenarios

    When migrating existing TLCs:

    • Register new TLCs with updated identifiers
    • Update session configurations to reference new TLCs
    • Verify sessions work with new TLC references
    • Remove old TLC registrations after successful migration

    Note: Since TLC information is immutable, migration requires creating new TLC registrations rather than updating existing ones.

    For complete API specifications and interactive testing, see the API Reference.
